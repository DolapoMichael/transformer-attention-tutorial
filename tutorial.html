1. Why Attention?

Before transformers, sequence models like RNNs and LSTMs were the default for natural language tasks. They process text token by token — left to right — compressing everything seen so far into a single fixed-size hidden state. This creates two well-known problems:

⚠ The RNN bottleneck

Long-range forgetting: By token 50, the model has largely overwritten information from token 1. Sentences like "The trophy didn't fit in the suitcase because it was too big" — does "it" refer to the trophy or the suitcase? A model needs to look far back.

Sequential bottleneck: Each step depends on the previous one — you can't parallelise training. This makes RNNs slow to scale.

The landmark 2017 paper "Attention is All You Need" (Vaswani et al.) proposed an architecture built entirely on attention — no recurrence, no convolution. Every token can directly interact with every other token in a single, fully parallelisable step.

✅ What attention gives us

Direct, learnable connections between any two positions — regardless of distance. The model can learn "when processing the word it, attend strongly to cat from 8 tokens ago."

2. The Query, Key, Value Intuition

Attention is often explained through the analogy of a soft database lookup. When you look something up in a dictionary, you have a search term, the dictionary has labelled entries, and each entry has content. Attention works the same way — but softly: instead of retrieving one exact match, it blends information from all entries, weighted by how relevant each one is.

Q
Query
What am I looking for? The current token "asking a question" of all others.
K
Key
What do I offer? Every token "advertising" how relevant it is to any query.
V
Value
What do I actually return? The information passed on if a match is found.

In practice, Q, K, and V are all computed from the same input — just with different learned weight matrices. This is called self-attention: each token attends to all others, including itself.

3. Scaled Dot-Product Attention

The full attention operation from Vaswani et al. is defined as:

Attention(Q, K, V) = softmax( QKᵀ / √d_k ) · V
Where d_k is the key dimension — used to prevent large dot products

This is computed in five steps:

1
Dot product scores Compute QKᵀ — a matrix of raw similarity scores between every query and every key. Shape: (seq_len × seq_len).
2
Scale by √d_k Without this, large values of d_k cause the dot products to grow, pushing the softmax into very flat or very peaked regions — both hurt gradient flow.
3
Optional masking In decoders, future positions are masked to -∞ so the model can't "look ahead" during training.
4
Softmax Converts scores to a probability distribution over positions — each row sums to 1. These are the attention weights.
5
Weighted sum of values The attention weights are used to blend the value vectors. The result is a new representation of each token, informed by all others.

4. Multi-Head Attention

A single attention head can learn only one type of relationship between tokens at a time. Multi-head attention runs h attention operations in parallel, each with its own learned Q, K, V projections:

MultiHead(Q,K,V) = Concat(head₁, ..., headₕ) Wᴼ
where headᵢ = Attention(Q Wᵢᴼ, K Wᵢᴷ, V Wᵢᵛ)
Each head projects into a reduced dimension d_k = d_model / h

Each head independently learns to focus on different aspects of the input. In practice, different heads have been observed to specialise — one might track syntactic dependencies, another coreference (like "it" → "cat"), another positional proximity.

Critically, because each head works in a lower-dimensional space (d_model / h), the total computational cost remains the same as a single full-dimension head — but with richer representational capacity.

Property Single-Head Multi-Head
Types of relationships learned One at a time h in parallel
Dimension per head d_model d_model / h (smaller)
Total computation Roughly equivalent
Representational expressiveness Lower Higher
Empirical performance Weaker Stronger (see Figure 8)

5. Positional Encoding

Attention is permutation-invariant — shuffling the order of tokens produces the exact same attention weights. This means the model has no built-in sense of word order, which is clearly important for language. Vaswani et al. address this by adding a positional encoding vector to each token embedding before attention is applied.

The sinusoidal encoding uses alternating sine and cosine functions:

PE(pos, 2i) = sin(pos / 10000^(2i / d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i / d_model))
pos = position in sequence; i = dimension index

This design was chosen because for any fixed offset k, the encoding of position pos + k can be expressed as a linear function of pos — making it easy for the model to learn relative positions. See Figure 4 in the notebook for a visualisation of the resulting encoding matrix.

6. Demo: Sentiment Analysis on IMDB

To demonstrate attention in action, we build a transformer encoder and train it on the IMDB movie review dataset — 50,000 labelled reviews (positive/negative). We use a subset of 5,000 training samples to keep training time manageable.

Architecture

The model consists of:

  1. A token embedding layer (vocabulary size 10,000, d_model = 128)
  2. Sinusoidal positional encoding
  3. Two stacked multi-head self-attention layers (4 heads each), each followed by a feed-forward sub-layer and layer normalisation
  4. Mean pooling over the sequence dimension
  5. A linear classifier head (2 outputs: positive / negative)

Visualising What the Model Attends To

7. Results: Single-Head vs Multi-Head

We train three variants of the same model — differing only in the number of attention heads — and compare test accuracy on the held-out IMDB test set after 5 epochs:

ModelHeadsd_k per headTest Accuracy
Single-head1128 ~79–81%
Multi-head (4)432 ~83–86%
Multi-head (8)816 ~84–87%

Multi-head attention consistently outperforms single-head, with diminishing returns beyond 4–8 heads for this task size. See Figure 8 in the notebook for the full training curve comparison.

💡 Key finding
The performance gap between 1 and 4 heads is much larger than between 4 and 8 heads — suggesting that the diversity of relational patterns captured by the first few heads provides the biggest gain.

8. Key Takeaways

  1. Attention solves the sequential bottleneck of RNNs — every token directly and simultaneously attends to every other token.
  2. Scaled Dot-Product Attention computes Q·Kᵀ similarity scores, scales by √d_k to stabilise gradients, then uses softmax weights to blend value vectors.
  3. Multi-head attention runs h attention functions in parallel at reduced dimension — each head learns a different type of relationship, for the same computational cost.
  4. Positional encoding is necessary because attention is order-agnostic; sinusoidal encodings inject position information that models can generalise from.
  5. Attention weights are interpretable — we can inspect them to understand what patterns the model relies on, a property not available in most deep learning architectures.
